# v1.2.0 Release Candidate Checklist

Use this checklist before promoting the client MVP usability milestone toward a formal `v1.2.0` release candidate.

---

## 1. Scope sanity

- [ ] v1.2.0 scope is limited to client MVP usability
- [ ] no speculative protocol expansion is bundled into the RC
- [ ] milestone notes and PR scope match the shipped code

## 2. Branch / CI state

- [ ] release branch content is based on the intended `main` candidate
- [ ] `main` is green before tagging
- [ ] no known blocker is open against packaging / runtime / diagnostics
- [ ] latest relevant GitHub Actions packaging/release runs are reviewed

## 3. Client quality gates

- [ ] `flutter analyze` passes
- [ ] client test suite passes
- [ ] profile CRUD / import / export flows are manually spot-checked
- [ ] secure storage flow is manually spot-checked
- [ ] runtime launch / stop / status / log flow is manually spot-checked
- [ ] diagnostics export flow is manually spot-checked

## 4. Desktop artifact smoke checks

Run:

```bash
scripts/smoke-check-client-artifacts.sh artifacts --platforms linux,windows,macos
```

If Android is part of the candidate:

```bash
scripts/smoke-check-client-artifacts.sh artifacts --platforms linux,windows,macos --expect-android
```

Checklist:
- [ ] Linux `.deb` install/extract smoke passes
- [ ] Linux bundle extract/start smoke passes, or is explicitly marked headless-skipped when no GUI/display runner is available
- [ ] Windows `.zip` contains expected `.exe`
- [ ] macOS `.app.zip` contains expected app bundle structure
- [ ] Android `.apk` structure is verified when included

## 5. Checksum verification

Run:

```bash
scripts/verify-artifact-checksums.sh artifacts
```

Checklist:
- [ ] every packaged client artifact has a matching `.sha256`
- [ ] checksum verification passes locally or in CI validation output
- [ ] artifact names and checksum names align cleanly

## 6. Release notes / docs

- [ ] release notes scope is drafted
- [ ] relevant docs are updated
- [ ] known limitations are stated honestly
- [ ] first-run / support burden is acceptable for the target audience

## 7. Final go / no-go

- [ ] no blocker remains for client install / start / basic profile usage
- [ ] rollback posture is understood
- [ ] candidate is acceptable for tag / publish

---

## Related commands

```bash
flutter analyze
flutter test
./scripts/run-client-runtime-smoke.sh
./scripts/verify-artifact-checksums.sh artifacts
./scripts/smoke-check-client-artifacts.sh artifacts --platforms linux,windows,macos
```
