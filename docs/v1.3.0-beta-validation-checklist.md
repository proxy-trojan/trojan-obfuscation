# v1.3.0 Beta Validation Checklist

Use this checklist before treating `v1.3.0-desktop-beta` as a promotable desktop beta candidate.

---

## 1. Source + branch sanity

- [ ] candidate branch is `feature/v1.3.0-desktop-beta` (or its intended merge candidate)
- [ ] scope matches the desktop-beta milestone rather than protocol expansion work
- [ ] latest local commit set is intentional and documented

## 2. Static quality gates

- [ ] `flutter analyze` passes
- [ ] `flutter test` passes
- [ ] no known blocker remains for settings / lifecycle / diagnostics / packaging surfaces

## 3. Desktop shell behavior

- [ ] window launches without immediate crash
- [ ] close/minimize/quit semantics match Settings and docs
- [ ] tray menu renders when platform support is present
- [ ] tray `Open` works
- [ ] tray `Connect` / `Disconnect` action gating matches current controller/profile state
- [ ] secondary app launch focuses primary instance or exits cleanly with no duplicate-window confusion

## 4. Profile + secure storage flow

- [ ] create profile manually
- [ ] edit/update profile successfully
- [ ] store Trojan password successfully
- [ ] selected profile is reflected consistently across Dashboard / Profiles / tray action label

## 5. Runtime smoke

- [ ] runtime mode is correct for the test path (`stubbed-local-boundary` or `external-runtime-boundary`)
- [ ] connect attempt transitions through understandable states
- [ ] disconnect clears runtime session state and config artifacts
- [ ] failure state produces useful summary and recovery action
- [ ] stale recovery behaves truthfully on relaunch when applicable

## 6. Diagnostics + support

- [ ] diagnostics preview renders
- [ ] diagnostics export succeeds on a real file-backed path
- [ ] support/problem-report overview reflects latest runtime failure state accurately

## 7. Packaging + update skeleton

- [ ] packaging snapshot export succeeds
- [ ] update channel is visible in Settings
- [ ] update-check stub records timestamp/status and does not pretend a real updater exists
- [ ] release metadata contract export includes current contract version

## 8. Docs + release posture

- [ ] `desktop-lifecycle-behavior.md` matches the shipped behavior
- [ ] `update-channel-skeleton.md` matches the shipped behavior
- [ ] known limitations are stated honestly for beta testers
- [ ] rollback posture is understood before any wider handoff
