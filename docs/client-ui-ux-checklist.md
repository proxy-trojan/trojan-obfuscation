# Client UI / UX Checklist

Use this checklist during the current UI validation phase.

## Menu / Navigation

- [ ] Top-level navigation only contains primary user destinations
- [ ] Advanced tools do not compete with the main workflow
- [ ] Navigation labels read like product language, not engineering labels
- [ ] A new user can identify where to start within a few seconds
- [ ] Settings and Advanced both feel like product pages, not control panels

## Layout

- [ ] The primary task is visually dominant
- [ ] Important state can be scanned quickly
- [ ] Secondary details are collapsed or visually demoted
- [ ] Pages do not feel crowded or overloaded

## Buttons

- [ ] Every page has one obvious main action
- [ ] Secondary buttons are visually quieter
- [ ] Dangerous actions are separated from routine actions
- [ ] Button labels describe user intent clearly

## Dialogs

- [ ] Dialog titles are simple and user-facing
- [ ] Forms are short and well-grouped
- [ ] Helper text explains uncertainty without overloading the user
- [ ] Confirmations are used for destructive actions only
- [ ] Dialog button labels match the product tone used on the main pages

## First-Run Guidance

- [ ] No-profile state tells the user what to do next
- [ ] Missing-password state tells the user what to do next
- [ ] Failed-connection state points the user to the right next tool
- [ ] The product supports a short experimental workflow

## Product Feel

- [ ] The UI feels like a product, not a control console
- [ ] A first-time user can understand the core workflow without explanation
- [ ] The UI rewards quick experimentation instead of punishing it
