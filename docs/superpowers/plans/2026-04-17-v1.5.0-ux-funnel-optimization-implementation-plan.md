# v1.5.0 UX Funnel Optimization 实现计划

> **面向 AI 代理的工作者：** 必需子技能：使用 superpowers:subagent-driven-development（推荐）或 superpowers:executing-plans 逐任务实现此计划。步骤使用复选框（`- [ ]`）语法来跟踪进度。

**目标：** 在不牺牲 runtime-truth 与 routing safety 口径的前提下，提升新用户首次连接成功率（FCSR）与老用户高频操作效率（HFE），并确保 SSR/STE 护栏不退化。

**架构：** 采用“漏斗优先 + 轻量数据闭环”策略：先固化事件字典与指标计算口径，再用最小体验改造包优化首次连接与高频路径，最后通过统一验证报告判定是否达标收口。所有改造遵循现有 controller/readiness/diagnostics 分层，不做大范围信息架构重写。

**技术栈：** Flutter/Dart、flutter_test、现有 LocalStateStore/DiagnosticsExport/Readiness 体系、Python 验证脚本（报告聚合）

```markdown
- denominator: unique users with `first_session_started`
- window: 30 minutes from first_session_started
- exclude: fallback/stub ready signals

- N_steps: median interaction steps per action
- R_rework: rate of action_rework_detected per action
```

```dart
class UxEvent {
  const UxEvent({required this.userId, required this.name, required this.at});
  final String userId;
  final String name;
  final DateTime at;
}

class RatioMetric {
  const RatioMetric({required this.numerator, required this.denominator})
      : value = denominator == 0 ? 0 : numerator / denominator;
  final int numerator;
  final int denominator;
  final double value;
}

class UxMetricService {
  RatioMetric computeFcsr(List<UxEvent> events) {
    final started = events.where((e) => e.name == 'first_session_started').map((e) => e.userId).toSet();
    final success = events
        .where((e) => e.name == 'runtime_session_ready_runtime_true')
        .map((e) => e.userId)
        .where(started.contains)
        .toSet();
    return RatioMetric(numerator: success.length, denominator: started.length);
  }
}
```
